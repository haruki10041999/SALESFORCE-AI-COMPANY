# Review Mode

コードファイルが提供された場合は、以下のレビューモードで動作してください。

## トリガー条件
- `filePaths` にファイルが含まれている
- 「レビュー」「確認」「チェック」という語が topic に含まれる

## レビュー時の発言構造
1. **概要評価**: 「このコードは〜という目的で〜の問題がある」を1文で
2. **重大度A（即修正）**: セキュリティ・governor limit・デプロイ不可の問題
3. **重大度B（要改善）**: バルク安全性・テスタビリティ・可読性の問題
4. **重大度C（推奨）**: 命名・コメント・スタイルの改善提案
5. **総合判定**: `APPROVE` / `REQUEST_CHANGES` / `NEEDS_DISCUSSION`

## レビューしてはいけないこと
- 機能要件の正しさ（仕様が不明な場合は確認する）
- ビジネスロジックの妥当性（product-manager に委ねる）

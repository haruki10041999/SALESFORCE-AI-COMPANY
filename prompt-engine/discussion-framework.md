# Discussion Framework

## 発言の構造
各エージェントの発言は以下の順序で構成してください。

1. **立場の表明**: 自分の専門領域から見た評価を1文で述べる
2. **根拠**: コード・数値・パターン名を根拠として示す
3. **懸念点**: リスクや考慮漏れを指摘する（ある場合）
4. **提案**: 具体的な次のアクションを示す

## 前提の扱い
- 前提は必ず明示する（「〜と仮定すると」）
- ブロッキングとなる不明点のみ質問する。些細な不明点は最善の仮定で進める

## トレードオフの扱い
- 複数の選択肢がある場合は「A案: メリット/デメリット / B案: メリット/デメリット」で対比する
- 最後に推奨案を1つ示す。推奨できない場合はその理由を述べる

## 会話の締め方
- 各ターンは「推奨する次のアクション」で終わる
- アクションは「誰が・何を・いつまでに」の形式で書く
- 全員の合意が取れた場合は「合意: 〜」と明示する

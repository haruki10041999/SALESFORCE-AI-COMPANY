# CRUD/FLS 強制

## 目的

CRUD と FLS チェックを適用し、Apex での不正なオブジェクト・項目アクセスを防止します。

## チェックリスト

- DML 前にオブジェクト権限（isCreateable、isUpdateable、isDeletable）を検証する。
- 読み取り・書き込みの項目権限を検証する。
- DML 前の入力サニタイズとレスポンス整形に Security.stripInaccessible を利用する。
- 動的フィールドセットやリフレクションベースのアクセスにガードを入れる。
- 権限不足ユーザーでの明示的なテストを追加する。

## 推奨パターン

- Read パス: 権限のある項目だけを取得し、DTO 出力をサニタイズする。
- Write パス: 受け取った sObject をサニタイズし、違反時は明確なエラーで早期失敗させる。
- Bulk パス: 可能な限りレコード単位ではなく、処理単位で権限チェックを行う。

## 注意シグナル

- CRUD/FLS ガードなしでサービスクラス内に DML がある。
- リクエスト payload から機密項目へ直接代入している。
- テストが管理者コンテキストでしか実行されていない。

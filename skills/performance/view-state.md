# UI / ビューステート パフォーマンス

## 目的

Salesforce フロントエンドコンポーネントのペイロード量と描画オーバーヘッドを削減します。

## チェックリスト

- リアクティブ状態の対象を最小化し、必要な項目のみ追跡する。
- ガード条件なしで enderedCallback に重い処理を書かない。
- グローバルリスナーは disconnectedCallback で解除する。
- 増分描画とメモ化した変換処理を優先する。
- 安全でない、または過剰な手動 DOM 操作を避ける。

## 注意シグナル

- 不要な再描画を引き起こす頻繁な state 再代入。
- 上限管理されていないイベントリスナー登録。
- 1 操作ごとのシリアライズ payload が大きすぎる。

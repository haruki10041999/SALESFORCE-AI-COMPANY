# SOQL 選択性レビュー

## 目的

大量データ環境でも、クエリの選択性と安定性を維持します。

## チェックリスト

- インデックス利用可能な条件（Id、External Id、選択的カスタムインデックス）を優先する。
- 前方ワイルドカード付き LIKE や広すぎる否定条件を避ける。
- OR 条件が多い場合は分割実行を検討し、述語を制御する。
- LIMIT/OFFSET の利用理由を明確化し、安全なページングを行う。
- 高ボリュームオブジェクトでは Query Plan を確認する。

## 注意シグナル

- 高カーディナリティオブジェクトに対する実質フルスキャン条件。
- 失敗後に条件を緩めるフォールバッククエリを繰り返している。
- Apex 側の後段フィルタに依存したクエリロジック。

# バルク処理パターン

## 目的

大量レコードでも制限値に抵触しないよう、Apex ロジックを効率的に設計します。

## チェックリスト

- ID を集約し、実行経路ごとのクエリ回数を最小化する。
- DML はループ外へ移し、可能であればオブジェクト種別単位でまとめる。
- Map / Set を使って繰り返し走査を避ける。
- 再帰呼び出しとトリガー重複実行を防止する。
- 非同期の分岐数を制御し、冪等性を確保する。

## 注意シグナル

- ネストしたループ内で SOQL / DML を実行している。
- トリガーハンドラが 1 レコードずつ処理している。
- 1 件の入力レコードに対して Queueable を複数回 enqueue している。
